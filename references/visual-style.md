# 共享视觉风格

适用于三个 Skill 的默认小红书图文视觉规范。

## 基础规则

- 画布：3:4 竖版，推荐 1080 x 1440。
- 背景：纯白，不使用渐变、纸张纹理、复杂背景或拟物场景。
- 线条：黑色手绘线稿，允许轻微自然抖动。
- 强调色：少量红、蓝、橙，只用于关键动作、警告、路径、状态或结论。
- 角色：黑色手绘小人，白点眼睛，细手细脚，表情可困惑、专注或轻微惊讶。
- 留白：保留大面积空白，让手机缩略图也能看清主动作。
- 每页：只表达一个核心认知点。

## 角色要求

角色必须参与核心技术动作，不能只站在角落装饰。

允许的动作：

- 观察、操作、输入、拖拽、连接、断开
- 搬运、替换、挑战、进入、离开、恢复
- 排队、等待、选择、确认、阻塞、释放
- 拿着命令纸条、打开终端房间、推动候选词卡片

## 好的技术隐喻

- 蓄水池抽样：新数字挑战当前候选座位。
- tmux session：服务器里的终端房间，用户离开后任务继续运行。
- SUG：搜索框下方伸出候选词卡片，经过防抖、请求、排序后展示。
- Toast：操作成功后冒出短暂提示纸条，不挡住用户继续操作。

## 避免

- 16:9 横版默认图。
- 商业 PPT 流程图。
- 写实摄影、光滑 3D、赛博服务器机房。
- 大段中文直接生成在图片中。
- 角色只是站在笔记本旁边。
- 无意义齿轮、代码雨、漂浮图标。
- 每页构图重复。
